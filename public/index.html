How it worked — step by step

  1. Salt generated client-side: Your browser created a random 256-bit salt.
  2. PBKDF2 derivation: Your password was stretched using PBKDF2-SHA256 (100 000 iterations) to produce a verifier. The password itself was discarded immediately.
  3. Server challenge: The server issued a fresh one-time random 256-bit challenge.
  4. HMAC proof: Your browser computed HMAC-SHA256(verifier, challenge) entirely locally.
  5. Verification: The server recomputed the same HMAC using the stored verifier and confirmed the proof matched — without ever seeing your password.
